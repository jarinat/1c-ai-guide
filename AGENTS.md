# AGENTS.md

Этот репозиторий является каноническим источником правил разработки на 1С.

## Назначение
Используй этот репозиторий для:
- написания ТЗ в нейросетевой форме;
- генерации нового кода на 1С;
- ревью кода;
- поиска ошибок;
- проверки соответствия правилам разработки.

## Порядок чтения
1. `README.md`
2. `docs/00-overview/how-to-read-this-guide.md`
3. `docs/01-standards/index.md`
4. `docs/02-playbooks/index.md`
5. тематический раздел, относящийся к задаче

## Приоритет правил
При конфликте применяй следующий порядок:
1. Явное исключение
2. Обязательное правило
3. Запрещённая практика
4. Рекомендуемая практика

## Правила поведения
- Не выдумывай правила, которых нет в репозитории.
- Если правило не найдено, явно укажи, что используешь общее инженерное суждение.
- Перед генерацией кода сначала определи, какие разделы стандартов относятся к задаче.
- Перед ревью сначала определи, какие обязательные и запрещённые практики применимы.
- Для ТЗ сначала используй playbook `docs/02-playbooks/write-spec/`.
- Для генерации кода сначала используй playbook `docs/02-playbooks/generate-code/`.
- Для ревью сначала используй playbook `docs/02-playbooks/review-code/`.
- Для поиска ошибок сначала используй playbook `docs/02-playbooks/find-bugs/`.

## Технологические рамки
Считай базовыми условиями:
- 1С:Предприятие 8.3.x
- управляемые формы
- разработка в EDT и конфигураторе
- обязательное использование БСП
- конфигурации типовые и самописные, но на базе БСП
