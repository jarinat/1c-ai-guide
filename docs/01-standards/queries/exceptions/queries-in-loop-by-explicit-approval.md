# QRY-EXCEPTION-001. Запросы в цикле по явному разрешению пользователя

## Название исключения
Использование запросов в цикле допускается только после явного разрешения пользователя.

## К какому правилу относится
[QRY-FORBIDDEN-001. Не выполнять запросы в цикле по умолчанию](../forbidden/no-queries-in-loop.md)

## Когда допускается
Исключение может быть применено только в двух случаях:
1. Пользователь прямо потребовал использовать запросы в цикле. Такое указание считается явным разрешением.
2. AI считает, что без запросов в цикле решение технически неизбежно либо они дают существенный выигрыш по архитектурной простоте, читаемости, стоимости поддержки или сроку реализации.

Во втором случае AI обязан сначала явно спросить разрешение пользователя, описать ожидаемый выигрыш и объяснить, почему решение без запросов в цикле хуже в данном контексте.

## Риски
Даже согласованный запрос в цикле может ухудшить производительность, масштабируемость и предсказуемость поведения. Исключение нельзя применять молча или как решение по умолчанию.

## Как документировать
Если пользователь ответил `да`, AI должен явно зафиксировать это в ТЗ, архитектурном описании, комментарии к решению или результате анализа: использование запросов в цикле разрешено пользователем для данного случая.

Если пользователь ответил `нет`, AI не должен использовать запросы в цикле и обязан предложить алгоритм или архитектуру без них.

Если исключение согласовано и цикл неизбежен, по возможности выноси создание объекта `Запрос` и текст запроса за пределы цикла, а внутри цикла меняй только параметры.
