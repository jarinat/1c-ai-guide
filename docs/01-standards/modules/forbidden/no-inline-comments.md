# MOD-FORBIDDEN-001. Не добавлять комментарии в код без явной необходимости

## Статус
Проверено

## Уровень
Запрещённая практика

## Область
Модули

## Краткое правило
Не добавляй комментарии в тело процедур и функций. Код должен быть самодокументирующимся. Допустимы только комментарии-описания экспортных методов (по стандарту 1С) или комментарии по явной просьбе пользователя.

## Зачем это нужно
Комментарии в теле кода быстро устаревают и начинают противоречить реальному поведению. Самодокументирующийся код (понятные имена переменных, процедур, небольшие функции с одной ответственностью) надёжнее комментариев. Избыточные комментарии создают визуальный шум и затрудняют чтение.

## Как применять
1. При написании кода не добавляй поясняющие комментарии внутри процедур и функций.
2. Вместо комментария — используй говорящее имя переменной или выдели блок в отдельную функцию с понятным именем.
3. Экспортные процедуры и функции **обязательно** документируй комментарием-описанием по стандарту 1С (назначение, параметры, возвращаемое значение).
4. При рефакторинге существующего кода — удаляй устаревшие комментарии.

## Как проверять в review
- Проверяй, что внутри тела процедур нет поясняющих комментариев (кроме редких случаев неочевидной бизнес-логики).
- Проверяй, что все экспортные методы имеют комментарий-описание.
- Если комментарий дублирует то, что очевидно из кода — требуй удаления.

## Пример правильно
```bsl
// Ищет строки ТЧ Товары по номенклатуре и характеристике.
//
// Параметры:
//   ДокументОбъект - ДокументОбъект - документ для поиска
//   НоменклатураСсылка - СправочникСсылка.Номенклатура
//   ХарактеристикаНаименование - Строка
//
// Возвращаемое значение:
//   Массив - найденные строки ТЧ
//
Функция НайтиСтрокиТЧ(ДокументОбъект, НоменклатураСсылка,
        ХарактеристикаНаименование) Экспорт

    ОтборНоменклатура = Новый Структура("Номенклатура", НоменклатураСсылка);
    СтрокиПоНоменклатуре = ДокументОбъект.Товары.НайтиСтроки(ОтборНоменклатура);

    СтрокиКандидаты = Новый Массив;
    Для Каждого СтрокаТЧ Из СтрокиПоНоменклатуре Цикл
        НаименованиеХар = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(
            СтрокаТЧ.Характеристика, "Наименование");
        Если НаименованиеХар = ХарактеристикаНаименование Тогда
            СтрокиКандидаты.Добавить(СтрокаТЧ);
        КонецЕсли;
    КонецЦикла;

    Возврат СтрокиКандидаты;

КонецФункции
```

## Пример неправильно
```bsl
Функция НайтиСтрокиТЧ(ДокументОбъект, НоменклатураСсылка,
        ХарактеристикаНаименование)

    // Сначала отбираем строки по номенклатуре
    ОтборНоменклатура = Новый Структура("Номенклатура", НоменклатураСсылка);
    СтрокиПоНоменклатуре = ДокументОбъект.Товары.НайтиСтроки(ОтборНоменклатура);

    // Фильтруем по наименованию характеристики
    СтрокиКандидаты = Новый Массив;
    Для Каждого СтрокаТЧ Из СтрокиПоНоменклатуре Цикл
        // Получаем наименование через БСП
        НаименованиеХар = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(
            СтрокаТЧ.Характеристика, "Наименование");
        // Проверяем совпадение
        Если НаименованиеХар = ХарактеристикаНаименование Тогда
            СтрокиКандидаты.Добавить(СтрокаТЧ);
        КонецЕсли;
    КонецЦикла;

    Возврат СтрокиКандидаты;

КонецФункции
```

## Исключения
- Комментарии-описания экспортных процедур и функций (по стандарту 1С) — обязательны.
- Комментарий по явной просьбе пользователя.
- Редкие случаи неочевидной бизнес-логики, которую невозможно выразить через именование.

## Источники
- Практический опыт разработки на 1С
