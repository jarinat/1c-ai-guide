# DAO-REC-001. Приведение типов данных из Excel перед использованием в запросах

## Статус
Проверено

## Уровень
Рекомендуемая практика

## Область
Данные и объекты

## Краткое правило
При чтении Excel через `ТабличныйДокумент` + `ПостроительЗапроса` всегда приводи типы колонок к целевым типам БД перед использованием в запросах или сравнениях.

## Зачем это нужно
`ТабличныйДокумент.Прочитать()` с `СпособЧтенияЗначенийТабличногоДокумента.Значение` автоматически определяет тип каждой ячейки. Числовые значения (артикулы, номера документов, размеры обуви) прочитываются как тип `Число`, даже если в БД они хранятся как `Строка`. Это ломает JOIN'ы и сравнения в запросах: `"12345" <> 12345`, `"42.5" <> 42.5`.

Дополнительные проблемы:
- Форматирование чисел зависит от локали: `12345` может стать `"12 345"` вместо `"12345"`
- Дробные числа: `42.5` может стать `"42,5"` вместо `"42.5"` (запятая вместо точки)
- Даты из Excel могут содержать время (`02.07.2025 17:49:10`), которое мешает сравнению с полем типа `Дата`

## Как применять
1. После чтения Excel через `ПостроительЗапроса.Выполнить().Выгрузить()` добавь этап приведения типов.
2. Для каждой колонки, которая будет использоваться в запросе, создай типизированную колонку с нужным `ОписаниеТипов`.
3. В одном обходе заполни все типизированные колонки, затем удали старые и переименуй новые.
4. Используй правильные форматные строки:
   - Числовой артикул/номер → строка: `Формат(Значение, "ЧГ=")` (без разделителей групп)
   - Дробное число → строка с точкой: `Формат(Значение, "ЧРД=.; ЧГ=")` (для размеров обуви и т.п.)
   - Строка → число: присвоение в типизированную колонку с `ОписаниеТипов("Число")`
   - Дата с временем → дата: присвоение в колонку с `КвалификаторыДаты(ЧастиДаты.ДатаВремя)`

## Как проверять в review
- Проверяй, что между чтением Excel и использованием данных в запросе есть этап приведения типов.
- Если колонка Excel используется в JOIN или сравнении со строковым полем БД — убедись, что она приведена к строке.
- Проверяй форматную строку: `"ЧГ="` обязателен для артикулов и номеров (иначе появятся пробелы-разделители), `"ЧРД=."` для дробных значений с точкой.

## Пример правильно
```bsl
// Добавляем типизированные колонки
ДанныеExcel.Колонки.Добавить("нАртикул", Новый ОписаниеТипов("Строка",
    , Новый КвалификаторыСтроки(50)));
ДанныеExcel.Колонки.Добавить("нХарактеристика", Новый ОписаниеТипов("Строка",
    , Новый КвалификаторыСтроки(150)));

Для Каждого Строка Из ДанныеExcel Цикл
    // Артикул: число -> строка без разделителей групп
    ЗначениеАртикул = Строка.НоменклатураАртикул;
    Если ТипЗнч(ЗначениеАртикул) = Тип("Число") Тогда
        Строка.нАртикул = Формат(ЗначениеАртикул, "ЧГ=");
    Иначе
        Строка.нАртикул = СокрЛП(ЗначениеАртикул);
    КонецЕсли;

    // Характеристика (размер обуви 42.5): число -> строка с точкой
    ЗначениеХар = Строка.Характеристика;
    Если ТипЗнч(ЗначениеХар) = Тип("Число") Тогда
        Строка.нХарактеристика = Формат(ЗначениеХар, "ЧРД=.; ЧГ=");
    Иначе
        Строка.нХарактеристика = СокрЛП(ЗначениеХар);
    КонецЕсли;
КонецЦикла;

// Заменяем колонки
ДанныеExcel.Колонки.Удалить(ДанныеExcel.Колонки.Найти("НоменклатураАртикул"));
ДанныеExcel.Колонки.Найти("нАртикул").Имя = "НоменклатураАртикул";
```

## Пример неправильно
```bsl
// Читаем Excel и сразу передаём в запрос — типы не приведены!
ТаблицаДанных = ПостроительЗапроса.Результат.Выгрузить();

Запрос = Новый Запрос;
Запрос.Текст = "ВЫБРАТЬ ... ИЗ &ДанныеExcel КАК Данные
    ЛЕВОЕ СОЕДИНЕНИЕ Справочник.Номенклатура КАК Ном
    ПО Данные.НоменклатураАртикул = Ном.Артикул";
// Артикул из Excel = Число(12345), в БД = Строка("12345") -> JOIN не сработает
Запрос.УстановитьПараметр("ДанныеExcel", ТаблицаДанных);
```

## Исключения
Если все значения колонки заведомо строковые (текстовые наименования без чисел), приведение типов не обязательно. Но безопаснее всегда приводить.
