# INT-REC-001. Паттерн загрузки Excel через временное хранилище

## Статус
Проверено

## Уровень
Рекомендуемая практика

## Область
Интеграции

## Краткое правило
Для загрузки Excel-файлов в управляемых формах используй паттерн: клиент помещает файл во временное хранилище через `НачатьПомещениеФайлаНаСервер`, сервер читает через `ТабличныйДокумент` + `ПостроительЗапроса`.

## Зачем это нужно
В тонком клиенте сервер не имеет прямого доступа к файловой системе клиента. Файл необходимо передать через временное хранилище. Чтение через `ТабличныйДокумент` (а не COM-объект Excel) работает на любом сервере без установленного MS Office.

## Как применять

### Клиентская часть

1. Проверить, что путь к файлу заполнен.
2. Вызвать `НачатьПомещениеФайлаНаСервер` с callback-описанием оповещения.
3. В callback получить адрес файла из `ОписаниеПомещенногоФайла.Адрес`.
4. Вызвать серверную процедуру обработки.

```bsl
&НаКлиенте
Процедура ПоместитьФайлИОбработать(РежимПроверки)
    Если НЕ ЗначениеЗаполнено(ПутьКФайлу) Тогда
        ПоказатьПредупреждение(, "Укажите путь к файлу Excel.");
        Возврат;
    КонецЕсли;

    ДопПараметры = Новый Структура("РежимПроверки", РежимПроверки);
    Оповещение = Новый ОписаниеОповещения("ПоместитьФайлЗавершение",
        ЭтотОбъект, ДопПараметры);

    // Параметры: Оповещение, , , , ПутьКФайлу, УникальныйИдентификатор
    // Параметры 2-4 (оповещения о ходе) пропускаются
    НачатьПомещениеФайлаНаСервер(Оповещение, , , , ПутьКФайлу,
        УникальныйИдентификатор);
КонецПроцедуры

&НаКлиенте
Процедура ПоместитьФайлЗавершение(ОписаниеПомещенногоФайла,
        ДополнительныеПараметры) Экспорт
    Если ОписаниеПомещенногоФайла = Неопределено
            ИЛИ ОписаниеПомещенногоФайла.ПомещениеФайлаОтменено Тогда
        Возврат;
    КонецЕсли;

    АдресФайла = ОписаниеПомещенногоФайла.Адрес;
    ОбработатьФайлНаСервере(АдресФайла,
        ДополнительныеПараметры.РежимПроверки);
КонецПроцедуры
```

### Серверная часть: чтение Excel

1. Получить двоичные данные из хранилища.
2. Записать во временный файл (`.xlsx`).
3. Прочитать через `ТабличныйДокумент` с `СпособЧтенияЗначенийТабличногоДокумента.Значение`.
4. Переименовать заголовки первой строки в программные имена колонок (по номеру позиции).
5. Выгрузить через `ПостроительЗапроса` в `ТаблицуЗначений`.
6. Удалить временный файл.

```bsl
&НаСервереБезКонтекста
Функция ПрочитатьФайлExcel(АдресФайла, НомерПервойСтроки)

    ДанныеФайла = ПолучитьИзВременногоХранилища(АдресФайла);
    ФайлНаДиске = ПолучитьИмяВременногоФайла("xlsx");
    ДанныеФайла.Записать(ФайлНаДиске);

    Попытка
        ТабДок = Новый ТабличныйДокумент;
        ТабДок.Прочитать(ФайлНаДиске,
            СпособЧтенияЗначенийТабличногоДокумента.Значение);
    Исключение
        УдалитьФайлы(ФайлНаДиске);
        ВызватьИсключение "Ошибка чтения файла Excel: "
            + ОписаниеОшибки();
    КонецПопытки;

    // Переименование заголовков по позиции
    СоответствиеКолонок = СоответствиеКолонокExcel();
    Для К = 1 По СоответствиеКолонок.Количество() Цикл
        ТабДок.Область(1, К, 1, К).Текст =
            СоответствиеКолонок.Получить(К);
    КонецЦикла;

    ПостроительЗапроса = Новый ПостроительЗапроса;
    ПостроительЗапроса.ИсточникДанных =
        Новый ОписаниеИсточникаДанных(ТабДок.Области[0]);
    ПостроительЗапроса.Выполнить();

    ТаблицаДанных = ПостроительЗапроса.Результат.Выгрузить();

    УдалитьФайлы(ФайлНаДиске);

    // Удаление лишних строк выше первой строки данных
    КоличествоЛишних = НомерПервойСтроки - 2;
    Для Сч = 1 По КоличествоЛишних Цикл
        Если ТаблицаДанных.Количество() > 0 Тогда
            ТаблицаДанных.Удалить(0);
        КонецЕсли;
    КонецЦикла;

    Возврат ТаблицаДанных;

КонецФункции
```

### Маппинг колонок

Соответствие номеров колонок Excel программным именам выносится в отдельную функцию:

```bsl
&НаСервереБезКонтекста
Функция СоответствиеКолонокExcel()
    Рез = Новый Соответствие;
    Рез.Вставить(1, "ИмяКолонки1");
    Рез.Вставить(2, "ИмяКолонки2");
    // ...
    Возврат Рез;
КонецФункции
```

## Как проверять в review
- Проверяй, что файл передаётся через временное хранилище, а не читается напрямую на клиенте.
- Проверяй наличие `УдалитьФайлы()` для временного файла (в том числе в ветке `Исключение`).
- Проверяй, что используется `СпособЧтенияЗначенийТабличногоДокумента.Значение` (а не `.Текст`).
- Проверяй наличие этапа приведения типов после чтения (см. [DAO-REC-001](../../data-and-objects/recommended/excel-type-casting.md)).

## Пример неправильно
```bsl
// Чтение напрямую на клиенте (не работает в тонком клиенте)
&НаКлиенте
Процедура Загрузить(Команда)
    ТабДок = Новый ТабличныйДокумент;
    ТабДок.Прочитать(ПутьКФайлу); // Ошибка: файл на клиенте, код на клиенте
КонецПроцедуры
```

## Исключения
В файловом режиме работы (клиент и сервер на одной машине) прямое чтение файла на сервере возможно, но паттерн с временным хранилищем универсальнее и работает в любом режиме.
