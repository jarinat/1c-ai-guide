# 1c-ai-guide

Центральный гид-репозиторий с правилами разработки на 1С для использования в:
- ChatGPT
- Claude Code
- Codex

## Цель

Репозиторий хранит:
- best practices по разработке на 1С;
- правила генерации нового кода;
- правила ревью и поиска ошибок;
- шаблоны ТЗ в нейросетевой форме;
- вспомогательные playbooks для AI-ассистентов.

## Основные принципы

1. Канонические правила хранятся только в `docs/01-standards/`.
2. Сценарии использования хранятся в `docs/02-playbooks/`.
3. `AGENTS.md` и `CLAUDE.md` — это файлы маршрутизации, а не дублирующая база знаний.
4. При конфликте правил приоритет определяется отдельно.

## Приоритет правил

Порядок применения правил:
1. Явное исключение для конкретного случая.
2. Обязательное правило.
3. Запрещённая практика.
4. Рекомендуемая практика.
5. Общие пояснения и примеры.

## Карта репозитория

- `docs/00-overview/` — базовое описание репозитория и шаблоны
- `docs/01-standards/` — правила по разработке
- `docs/02-playbooks/` — сценарии работы AI
- `docs/03-templates/` — шаблоны документов
- `docs/04-sources/` — источники и карта источников
- `docs/05-indexes/` — сводные индексы по правилам

## С чего начинать чтение

1. `docs/00-overview/how-to-read-this-guide.md`
2. `docs/01-standards/index.md`
3. `docs/02-playbooks/index.md`
4. нужный тематический раздел

## Подключение к проектам

Рекомендуется подключать этот репозиторий в рабочие проекты как submodule, например:

```bash
git submodule add <URL_РЕПОЗИТОРИЯ> .ai/guide
```

После этого локальные `AGENTS.md` и `CLAUDE.md` рабочего проекта должны ссылаться на `.ai/guide/` как на источник правил.

## Версионирование репозитория

Репозиторий использует версию в формате `YYYY.MM.DD.N`, например `2026.03.23.1`.

- `YYYY.MM.DD` — дата коммита.
- `N` — порядковый номер коммита в пределах этой даты.
- Каноническое значение хранится в файле `VERSION`.

Для обновления версии используй:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/bump-version.ps1 -Stage
```

Скрипт увеличивает счётчик для текущей даты или начинает его с `1`, если дата изменилась.

## Проверка перед коммитом

В репозитории есть отслеживаемый хук `.githooks/pre-commit`, который не даёт выполнить коммит, если:

- `VERSION` не добавлен в индекс;
- версия не соответствует формату `YYYY.MM.DD.N`;
- дата в версии не совпадает с текущей датой коммита;
- счётчик не увеличен относительно `HEAD`;
- для новой даты счётчик начинается не с `1`.

Чтобы включить хук локально, один раз выполни:

```powershell
git config core.hooksPath .githooks
```
